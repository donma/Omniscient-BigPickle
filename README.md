# Omniscient BigPickle — 全職設計視角

> 由 AI 模型 **BigPickle** 於 2026 年生成驅動 · 90 套專業網站樣板展示館

**全職設計視角**是一個大規模靜態網站樣板集合，由 90 套獨立設計的完整多頁網站樣板組成，橫跨 25 種版型與 15+ 產業類別。每一套樣板都是從零開始構思的設計作品，配備 12 頁完整內容，而非一般公版集合僅更換顏色與文字的廉價做法。

本專案完全以 **BigPickle** 模型進行設計生成與內容創作，展現 AI 在網頁設計領域所能達到的專業廣度與細節深度。

## 專案願景

在傳統網頁設計流程中，產出一套完整的多頁設計需要設計師、前端工程師、內容撰寫者協作數天甚至數週。**全職設計視角** 證明了 AI — 具體而言是 **BigPickle** 模型 — 能夠大規模產出高品質、多樣化、具備實際可用性的網站樣板，涵蓋從 Landing Page 到企業官網、從電商平台到知識中心的各種場景。

本專案不僅是樣板集合，更是一次設計實驗：探討 AI 是否能掌握版型節奏、色彩系統、資訊層級、響應式佈局以及產業語境等設計師日常面對的核心課題。

### 設計哲學

- **多樣性優先**：90 套樣板，每套保證與其他 89 套至少有 4 個結構差異
- **內容驅動**：每套 12 頁完整內容，非 Lorem Ipsum 填充
- **產業語境**：色彩、字體、語氣皆配合產業特性
- **可實踐性**：所有樣板皆為可直接使用的 HTML/CSS/JS

## 開發與擴展

### 新增一套樣板

1. 在 `templates/` 下建立 `template-091-company-name/`
2. 加入 12 個 HTML 頁面及對應資源
3. 在 `data/templates.json` 註冊樣板後設資料
4. 在 `data/layout-assignment.json` 登記版型分類
5. 產生預覽縮圖置於 `assets/img/previews/`

### 自訂展示站樣式

修改 `assets/css/gallery.css` 即可調整展示館的視覺風格。

編輯 `data/templates.json`，格式為 JSON 陣列，每個物件代表一套樣板：

```json
{
  "id": 1,
  "slug": "template-001-sumi-sushi",
  "name": "墨染壽司",
  "industry": "美容健康",
  "layout": "左右分割 Split Layout",
  "colorTheme": "黑金色",
  "description": "..."
}
```
